ph

Principled HTTP

A principled HTTP library for Scala.

Built on high-quality, principled libraries.

ph is backed by scalaz, Monocle, Argonaut, and Specs2 to provide a clean, safe, consistent, and purely functional API. Use lenses to navigate response types and traverse JSON (after this issue lands in Argonaut).

Lenses

ph uses the awesome Monocle library and enables you to navigate your HTTP responses by composition using lenses. If the response is JSON, you can traverse the JSON using the same syntax (at least you will be able to when argonaut#100 is implemented. 

object DisplayIP extends SafeApp {
  override def runc: IO[Unit] =
    for {
      ip <- http get "http://da.gd/ip"
      _ <- putStrLn(ip |-> body get)
    } yield ()
}

Client-agnostic

ph is built from the core to be agnostic to backend clients. As of right now, ph doesn't implement its own handling of the HTTP protocol, but instead provides a purely functional wrapper over clients such as Dispatch, HttpURLConnection, and Apache HttpClient, although as of this writing, HttpURLConnection is what is being used as the reference client. 

trait Client {
  def get(url: String): IO[Response[ByteString]]
